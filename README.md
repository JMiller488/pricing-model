# pricing-model

A regression-based pricing analysis that surfaces revenue leakage in transactional sales data.

## Background

This project rebuilds a piece of pricing analysis I originally delivered for an FMCG client. The goal is to use build an Average Sales Price (ASP) regression model on historical data, and measure individual transactions against this baseline. The aim is to surface customers whose pricing deviates from the baseline as candidates for further commercial review.

## Methods

Linear regression on one-hot encoded categorical features and a log-transformed numeric feature, fitted separately per period. Cook's distance via `statsmodels` for influence diagnostics. Winsorisation of percentage residuals for clean summary statistics. Customer-level aggregation of dollar-denominated leakage with a waterfall visualisation of the top contributors.

Built with `pandas`, `scikit-learn`, `statsmodels`, and `matplotlib`.

## What's in the notebook

The full analysis lives in [`notebooks/pricing_model.ipynb`](notebooks/pricing_model.ipynb). It walks through:

1. **A per-period regression** of ASP on product, state, and log-quantity, fitted separately for two years of sales data.
2. **A Cook's distance diagnostic** to identify and remove a small number of high-influence rows that were distorting the Y1 fit, with a discussion around whether removal is commercially justified.
3. **Residual analysis and winsorisation** to handle the heavy-tailed distribution of percentage price gaps without losing the underlying signals.
4. **Customer-level leakage quantification** showing that a small number of accounts dominate the total opportunity, with a waterfall chart of the top contributors.

## Running it

```bash
git clone https://github.com/JMiller488/pricing-model.git
cd pricing-model

python3 -m venv .venv
source .venv/bin/activate
pip install pandas numpy scikit-learn statsmodels matplotlib jupyter

python3 scripts/generate_synthetic.py
jupyter notebook notebooks/pricing_model.ipynb
```

The synthetic dataset is also committed under `data/`, so the notebook can be run directly without regenerating if preferred.

## A note on the data

The notebook runs on a synthetic dataset generated by [`scripts/generate_synthetic.py`](scripts/generate_synthetic.py). The real data was from a client engagement and not shareable. The synthetic generator deliberately reproduces the structural features of the original — state-level freight premiums, volume discounting, a small set of systematically underpriced customers — so the analytical story matches roughly what the real data revealed. Specific customer codes and dollar figures are illustrative.
